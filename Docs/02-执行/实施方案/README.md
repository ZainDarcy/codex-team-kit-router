# 实施方案索引

本目录只放中大型迭代、架构迁移、公共规则调整或高风险实现的方案包。小问答、小修复和低风险单文件改动不要在这里建方案。

## 使用规则

- 先在本索引登记方案包，再进入具体方案文件夹。
- 每个方案包用 `YYYY-MM-DD-主题/` 命名，并从包内 `README.md` 读起。
- 方案包只保留必要字段：目标场景、版本范围、明确不做、预计改动面、验收标准、迁移/回退风险、验证方式、参与角色、待批准问题。
- 用户批准必须指向具体方案名或版本；批准前只允许继续评审、补方案或更新进度。
- 进入写入阶段前，手动运行 `.codex/hooks/pre-implementation`；若失败，先补方案或批准状态。

## 防臃肿约束

- 不把聊天记录、完整子代理日志或长调研原文塞进方案包。
- 不复制 `项目规范.md`、`项目进度.md` 或行业包说明；只链接必要真相源。
- 行业专属示例留在对应 `.codex/agent-packs/`，不要写进通用方案索引。
