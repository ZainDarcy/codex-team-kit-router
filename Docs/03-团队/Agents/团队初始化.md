# 团队初始化

本文件定义模板接入目标项目后的团队初始化动作。模板仓库不固定团队成员人名；用户把 GitHub 地址发给目标项目里的 AI 后，由 AI 在目标项目初始化时随机生成成员显示名。

## 何时必须执行

- 第一次在目标项目使用本模板时。
- 用户把 `https://github.com/ZainDarcy/codex-team-kit-router` 或本模板 fork 地址发给 AI，并要求接入团队模板时。
- 第一次走 `Team` 路由或准备派发子代理前。
- `Docs/03-团队/Agents/团队名册.md` 状态仍为 `pending` 时。
- `.codex/agents/*.toml` 新增、删除或替换默认核心 agent 后。

## 初始化输入

初始化前，Codex 主线程读取或合并：

- `AGENTS.md`
- `Docs/README.md`
- `Docs/03-团队/开发团队.md`
- `.codex/agents/*.toml`
- 本文件

不要读取历史工作记录全文。

## 起名规则

- 按目标项目主要用户的语言习惯起名。
- 名字必须像真实团队成员的人名，不要用岗位名、功能名、诗化称号或工具名。
- 中文项目使用随机的二字或三字中文人名。
- 英文项目使用自然英文人名。
- 其他语言项目使用对应语言下自然的人名。
- 每个成员名字必须唯一。
- `.codex/agents/*.toml` 的 `name` 字段保持英文机器名，不随显示名变化。
- 不使用本模板文档中的示例名作为固定输出；每个目标项目都重新随机生成。

## 必须创建或更新

初始化必须完成：

1. 更新 `Docs/03-团队/Agents/团队名册.md`，把状态改为 `initialized`。
2. 为 `.codex/agents/*.toml` 中每个默认核心 agent 创建成员档案：

```text
Docs/03-团队/Agents/成员档案/{agent_name}-{随机显示名}.md
```

3. 每个成员档案必须基于 `_成员档案模板.md`，并写入：

- Agent 岗位
- 显示昵称
- 昵称语言
- 首次启用日期
- 适用任务
- 不适用任务
- 工作风格
- 项目理解
- 常见风险
- 适合继续派发的任务

4. 不创建初始工作记录；工作记录只在真实任务执行后创建。

## 初始化后检查

- `团队名册.md` 中每个默认核心 agent 都有一行。
- 每一行都有对应的成员档案文件。
- 没有重复显示名。
- 没有使用岗位名作为显示名。
- 名册和档案只写摘要，不粘贴长上下文。
