# Docs 导航

`Docs/` 按运行时用途分层，避免主线程为了找规则一次性读取全部文档。接入已有项目后，实际文档根目录以 `.codex/team-kit.toml` 的 `docs_root` 为准。

## 目录

- `01-项目/`：项目稳定规范、当前进度和迁移时需要保留的长期信息。
- `02-执行/`：AI 执行手册、文档路由和按需加载规则。
- `03-团队/`：团队结构、子代理工作记录、成员档案和交付模板。
- `03-团队/行业扩展包/`：可选行业扩展包说明，当前包含游戏行业包。

## 使用原则

- 简单任务只读 `AGENTS.md` 和用户点名文件。
- 普通项目任务读 `01-项目/` 和 `02-执行/` 的相关文件。
- 已初始化项目的日常团队或子代理任务读 `02-执行/Team运行卡.md`、`03-团队/开发团队.md` 和 `.codex/team/` 的相关文件。
- 接入或迁移任务先读 `.codex/team-kit.toml`、`03-团队/Agents/团队初始化.md` 和 `03-团队/行业扩展包/README.md`，确认 staging 目录、路径、真相源、初始化模式和行业扩展包选择。
- 不把 `03-团队/Agents/` 的历史记录整篇常驻注入主线程。
- 走团队流程后，至少写一份本波工作记录，按参与者更新 `03-团队/Agents/团队名册.md` 的最近任务和任务次数；成员档案只在出现新增可复用经验时更新。
